Localstorage

Note: From version 4 on, the LocalStorage package introduces an important breaking change. Instead of providing an interface depending on a given identifier, saving the state under that identifier, this package now only provides the raw localStorage or sessionStorage objects. It is in a sense no more than a wrapper around the window.localStorage and window.sessionStorage objects. It is now every application's responsibility to save the state in localStorage under appropriate identifiers, avoiding mixing up states from different application hosts. Alternatively, you may use one of the many available Angular libraries for binding the localStorage to redux and/or identifiers.
